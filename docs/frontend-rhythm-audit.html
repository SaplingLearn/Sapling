02 The crux — same token name, different value

The pre-auth surface re-declares semantic tokens inside the .landing-page scope. So identical class names render warm in-app and cool in pre-auth. globals.css:60–64, 135–149, 762–789

TokenApp shell (:root)Pre-auth (.landing-page)Effect
--border warm 42,39,31 cool slate 107,114,128 borders change temperature
--text-muted #6f6857 taupe #4b5563 slate secondary text shifts hue
--text #1a1814 warm-black #111827 cool-black even the ink differs
--shadow-* green-tinted 19,38,16 slate-tinted 15,23,42 shadows warm vs cool
--dur-fast / -slow 140 / 420ms 120 / 350ms same name, different speed
--r-* vs --radius-* --r-sm…--r-xl --radius-sm…-full duplicate names, same values

A component literally cannot move between zones without being re-tokenized. This is the structural fix that makes everything below stay fixed.

06 Documented inconsistencies

Severity: P0 breaks identity / first impression · P1 visible rhythm break · P2 systemic drift / rot · P3 polish.

P0

A · Five greens for "primary"

Forest, forest-bright, an un-matched glow green, plus the pre-auth "main" + "success" greens — and sage for accents. globals.css:21–25, 738–739, 1002 · app/page.tsx:783

FixOne primary + one bright hover; delete/alias --brand-primary/--brand-success; derive the glow from the same token.
P0

B · Button is a different species pre/post sign-in

Beta = rounded-full pill, 16px, infinite beta-glow + shimmer. App = rectangular, 6px, 13px, no motion. app/page.tsx:779–787 · globals.css:205, 1005

FixRe-skin the beta CTA to the app's button language; lose the infinite glow (a one-shot entrance is fine).
P1

C · Onboarding is color-correct but structurally orphaned

Uses app tokens ✓, but renders as a centered card in a radial-gradient void with no shell, hard-coded px spacing, and a bespoke type scale. screens/Onboarding.tsx:164, 171, 179, 268

FixAdopt --pad-* spacing + the shell type scale; render inside the app's left-aligned frame, not a void.
P1

D · Pending/waitlist is a product-identity whiplash

Beta = "join the movement" (pulsing glow). Two seconds later pending = bare centered flex, plain .btn, no card, no confirmation beat. pending/page.tsx:15–40

FixGive pending the app's surfaces + a brief "you're in ✓" moment so the glow resolves into something.
P1

E · The content box has 3+ treatments

App .card (16px) vs inline 24px warm-gradient modal (duplicated in beta + sign-in) vs card-in-void vs no box. globals.css:212 · app/page.tsx:975 · SignInModal.tsx:224

FixOne surface primitive; make the pre-auth "hero card" a named variant of .card, not an inline copy.
P1

F · Typography access pattern + scale diverges

App uses semantic classes (.h-serif) at a controlled scale; pre-auth uses utility classes (.font-playfair) with inline px at 700–800 weight. app/page.tsx:1022, 1070

FixDefine the display scale once; let pre-auth use the top of the same ramp, not a parallel one.
P2

G · Animation energy is inverted at the worst moment

The only persistent motion in the product lives in pre-auth: glow (∞), mesh blobs (10s ∞), card-float (∞), shimmer. The app is deliberately still. globals.css:831, 1005, 934–939

FixDial pre-auth motion to one or two finite moments so the app's calm feels earned.
P2

H · Two token namespaces guarantee drift

--r-* vs --radius-*, --ease vs --ease-out, --dur vs --dur-base, plus the value-shadowing above. globals.css:128–149 vs 779–789

FixOne namespace. Map pre-auth onto the app tokens and delete the duplicates. The structural root fix.
P2

I · Dead / duplicated implementations (rot)

OnboardingFlow.tsx (36 KB, unused) carries the old landing DNA and isn't imported by the active route. Beta + sign-in modals duplicate one card. components/OnboardingFlow.tsx

FixDelete OnboardingFlow.tsx; extract the shared pre-auth card.
P3

J · Modal-first for high-intent moments

Beta signup & sign-in are both modals over the landing — worth questioning whether "join the beta" deserves a dedicated on-brand page.

FixConsider a real page for beta signup instead of a glowing-pill-triggered overlay.

07 Remediation sequence

Do the structural work first — it's what keeps the visible fixes from drifting back.

Structural — do first
H + crux

Unify the token layer

One namespace, one set of semantic values. Stop re-declaring --border/--text/--shadow/--dur inside .landing-page. A cooler pre-auth palette should be a deliberate sub-theme, not an accidental shadow of the same names.

A

Collapse the greens

One primary, one bright, one derived glow. Alias or delete the rest.

Visible wins
B + E

Re-skin beta CTA + signup to the app's language

Lose the infinite glow; share one card primitive with sign-in.

C + D

Re-home onboarding + pending

Into the app's layout, spacing, and type — and add the missing "you're in" confirmation beat.

Cleanup
I

Delete dead code; extract the shared card

Remove OnboardingFlow.tsx; pull the duplicated pre-auth box into one component.

G

Normalize pre-auth motion

Finite, purposeful moments only.

08 What's already right

The active onboarding is token-disciplined (no hard-coded hex) — its problem is layout, not color. pending correctly uses app tokens — it just needs the app's surfaces. Glassmorphism was already removed (#102) with class names kept as solid. And the app shell itself is genuinely cohesive: a clear warm-editorial voice, one button family, a real spacing-token system. It's a strong source of truth to converge on.
