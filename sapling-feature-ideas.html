Sapling — Internal Document

Feature Ideas for University Students

A deep-dive into high-impact features to increase engagement, retention, and growth among the university student demographic.

Prepared March 2026  ·  Confidential
High Impact / High Excitement
📝

AI Practice Exams

Directly addresses exam anxiety — the #1 student stressor

Generate a full mock exam from any uploaded lecture notes, slides, or documents. The exam is timed, auto-graded, and each wrong answer comes with an AI-generated explanation tied back to the source material. Students can retake with reshuffled questions, track score history, and identify weak areas.

Key Capabilities

  • Multiple choice, short answer, and true/false question types
  • Configurable exam length and time limit
  • Per-question explanations tied to source docs
  • Score history and weak-topic identification
  • Re-take mode with reshuffled questions

Why It Works

  • Exam prep is the highest-urgency student activity
  • Builds on existing document upload + Gemini pipeline
  • Drives repeat uploads of new course material
  • Differentiated from basic flashcard apps
🎓

Grade Calculator

Simple, universally needed, daily active use driver

Students input their current grades and assignment weights per course, and Sapling calculates exactly what they need on remaining assignments or the final exam to reach their target grade. If a syllabus has already been uploaded, weights can be auto-populated. Supports both percentage-based and point-based grading schemas.

Key Capabilities

  • Auto-populate weights from uploaded syllabus
  • Scenario modeling ("what if I get an 80 on the final?")
  • GPA impact calculation
  • Drop-lowest-score support
  • Per-course grade dashboard

Why It Works

  • Every student Googles a grade calculator every semester
  • Extremely low build effort, high retention payoff
  • Encourages syllabus uploads (feeds other features)
  • Keeps students coming back throughout the semester
🏆

Competitive Study Rooms

Gamifies collaboration — turns studying social

Layer competitive mechanics on top of the existing study rooms. Room members can see each other's weekly study time, quiz scores, and knowledge graph mastery side-by-side. Weekly leaderboards reset every Monday. A live "study battle" mode lets two students race to answer the same quiz questions.

Key Capabilities

  • Weekly leaderboard (study time, quiz scores, mastery gain)
  • Side-by-side knowledge graph comparisons
  • Live "study battle" — real-time quiz races
  • Room XP / level system
  • Achievement badges (e.g. "First to master all nodes")

Why It Works

  • Friendly competition is a proven retention mechanic
  • Builds on existing rooms infrastructure
  • Encourages students to invite classmates (organic growth)
  • Gives study rooms a reason to come back to daily
✨

"Wrapped" Study Stats

Shareable recap — organic social media growth

At the end of each week and semester, Sapling generates a visually striking recap card: total hours studied, concepts mastered, top subject, longest streak, most-used study mode. Cards are designed to be shared — one tap copies a shareable image. Semester wrap-ups could go viral during finals season.

Key Capabilities

  • Weekly and semester recap cards
  • Shareable image export (PNG/link)
  • Animated reveal on first view
  • Comparison to previous weeks/semesters
  • Highlights: top subject, study streak, busiest day

Why It Works

  • Students love sharing achievements — see Duolingo, Wordle
  • Sapling branding on every shared card = free impressions
  • Finals season drops could trend on campus social media
  • Motivates consistent use to "earn" a better recap
Practical / Sticky

Pomodoro Focus Timer

Keeps users in the app — increases session depth

An integrated focus timer built into learning sessions. Students pick a topic, set a Pomodoro block (default 25 min / 5 min break), and Sapling tracks exactly what concept they studied during each block. At the end of the session, a breakdown shows time-per-concept, helping students see where their study time actually goes.

Key Capabilities

  • Customizable work/break intervals (25/5, 50/10, etc.)
  • Per-block topic tagging — ties into knowledge graph
  • Session summary: time per concept, total focused time
  • Ambient sound options (rain, lo-fi, white noise)
  • Do-not-disturb mode suggestion during active blocks
🎙️

Voice Mode

Study anywhere — removes the typing barrier

Allow students to speak to the AI tutor instead of typing. Voice input is transcribed and sent to the existing learning pipeline. The AI response can be read back via text-to-speech. This is transformative for commuters, students reviewing on the go, or anyone who finds typing in a study session disruptive to their flow.

Key Capabilities

  • Push-to-talk or continuous voice input
  • Real-time transcription display
  • Optional AI voice read-back (adjustable speed)
  • Works across all session modes (Socratic, Quiz, Teachback)
  • Hands-free mode for mobile
📸

Screenshot → Study Notes

Removes upload friction — captures in-the-moment learning

Students snap a photo of a whiteboard, slide projected in class, or handwritten notes. Sapling OCRs the image, formats it into clean structured notes, and automatically maps new concepts into the knowledge graph. A quick mobile capture flow means students can add to their library without leaving class.

Key Capabilities

  • Camera capture directly in the mobile web app
  • OCR → clean formatted notes with headings and bullets
  • Automatic concept extraction and knowledge graph update
  • Batch upload of multiple screenshots into one document
  • Edit/correct OCR output before saving
Viral / Community
🌐

Shareable Knowledge Graph

Visually unique — spreads Sapling organically

Let users share a snapshot or live link to their knowledge graph. The 3D graph is one of Sapling's most visually distinctive features — a shareable version posted to Reddit, Twitter, or a Discord study server would generate curiosity and sign-ups. Public graphs could also show anonymized class-wide mastery averages if study room members opt in.

Key Capabilities

  • Generate a shareable link or image snapshot of your graph
  • Privacy controls: share full graph, subject-only, or stats-only
  • Embed in personal websites or portfolios
  • Study room aggregate view — "our group's collective knowledge"
  • Optional public profile page showing graph + stats
⚡

Study Challenges

Friend-to-friend sharing loop — viral invite mechanic

Issue a challenge to a friend or study room: "Master these 10 concepts before Friday." Both participants track progress in real-time. The challenger gets notified when the challengee catches up or overtakes them. Challenges can be shared as invite links, pulling non-users into Sapling.

Key Capabilities

  • Create a challenge from any topic or concept set
  • Invite via link — works for non-users (creates sign-up prompt)
  • Real-time progress tracker for both parties
  • Deadline and win condition customization
  • Challenge history and win/loss record
LMS & Deadline Integration

Gradescope does not have an official public API. The most practical path to automatic deadline sync is integrating with the LMS platforms that host Gradescope assignments. Below is a realistic breakdown of each platform.

Canvas LMS

Official OAuth API

The most common US university LMS. Has a full REST API with OAuth 2.0 — same pattern as the existing Google Calendar integration. Gradescope assignments sync to Canvas automatically via LTI. Recommended first integration.

Google Classroom

Official OAuth API

Strong in K-12 and some smaller colleges. Google Classroom API is well-documented and straightforward. Good secondary target alongside Canvas.

Blackboard

Official API, but complex

Has a REST API with OAuth, but requires registration with each institution's Blackboard instance. Hard to self-serve — better as a later addition once there's institutional demand.

Gradescope (direct)

No official API

No public API exists. Unofficial scraping libraries exist but require storing user passwords and break with SSO, which most universities use. Not recommended.

Moodle

Official API

Common internationally and at some US institutions. Has a REST API but adoption is more fragmented — worth targeting if international expansion is planned.

Email Parsing

Workaround

Gradescope sends email notifications for new assignments. Users forward emails to a Sapling address; AI extracts the deadline. Fragile but no credentials required — good fallback for non-Canvas schools.
